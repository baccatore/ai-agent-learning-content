# 学習コンテンツ設計書：AIエージェント活用ガイド
# Learning Content Design: AI Agent Practical Guide

**作成日 / Date:** 2026-03-11

## 1. 概要 / Overview

非エンジニアのビジネスパーソンが、Claude CodeをAIエージェントとして業務に活用し、
GitHubで業務管理と成果物の検収を効率的に行えるようになるための学習コンテンツ。

Learning content for non-engineer business professionals to leverage Claude Code as an
AI agent for business tasks, with GitHub-based task management and deliverable review.

## 2. ターゲット / Target Audience

- **レベル**: IT感度が高い非エンジニア
- **前提知識**: SaaSツールの日常的使用、AIチャット(ChatGPT等)の経験あり
- **未経験領域**: CLI/ターミナル操作、Git/GitHub

## 3. 成果物 / Deliverables

| 成果物 | 説明 |
|--------|------|
| ワークショップ資料 | 講師ファシリテート前提の全5回シリーズ。講師ガイド＋参加者資料 |
| 自己学習教材 | 一人で読み進めて実践できるテキスト教材。全4モジュール |

**言語**: 日本語・英語の両方（i18n対応、将来の多言語展開を想定）

## 4. カリキュラム構成 / Curriculum Structure

### ワークショップ（全5回シリーズ）

| Session | テーマ | 目的 |
|---------|--------|------|
| 0 | 体験 — AIエージェントの威力を実感する | GitHub上で即座にAIエージェントが動く様子を体験。「すごい」＋「理解しないとまずい」の動機づけ |
| 1 | 基礎 — AIエージェントを理解し、動かしてみる | AIチャットとの違いを理解し、Claude Codeを自分で操作できるようになる |
| 2 | 実践 — 業務タスクをエージェントに任せる | デスクトップリサーチ・提案資料作成を実際に行う |
| 3 | 実践応用 — データ分析とさらなるユースケース | データ分析・可視化＋追加ユースケースの実践 |
| 4 | 運用 — GitHubで業務を管理する | Issue/PRベースのタスク管理、チーム連携ワークフロー |

### 自己学習教材（全4モジュール）

| Module | テーマ | 内容 |
|--------|--------|------|
| 1 | 基礎 | AIエージェントの概念、セットアップ、基本操作、プロンプティング基礎 |
| 2 | 実践スキル | リサーチ、文書作成、データ分析の各業務での活用方法 |
| 3 | 応用ユースケース | 追加のビジネスユースケース、複合ワークフロー |
| 4 | 運用 | GitHub入門、タスク管理、チーム連携、ベストプラクティス |

## 5. Session 0 設計詳細

**コンセプト**: 講師が事前にGitHubリポジトリとClaude Code環境を準備。
参加者はGitHub上でIssueを作成するだけで、AIエージェントが自動的にタスクを実行し、
PRとして成果物を提出する一連の流れを体験する。

**体験フロー**:
1. 講師がリポジトリとGitHub Actionsの仕組みを簡単に紹介
2. 参加者がIssueに業務タスク（リサーチ依頼等）を記入
3. Claude Codeがタスクを実行、PRを自動作成
4. 参加者がPR上で成果物をレビュー・コメント
5. 振り返り：「今何が起きたか」の解説

**期待効果**:
- 「こんなことができるのか」という驚きと興奮
- 「これを理解しないと取り残される」という健全な危機感
- 以降のセッションへの強い学習動機

## 7. 設計方針

- **YAGNI**: 必要最小限の内容に絞る。将来の拡張は後で判断
- **実践重視**: 概念説明は最小限に、手を動かす時間を最大化
- **安全性**: サプライチェーン攻撃やプロンプトインジェクションへの注意喚起を含む
- **i18n**: 言語フォルダ分離で多言語展開に対応
